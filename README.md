# 550W

> 一个让 AI Agent 学会自我进化的系统级引擎。装上即生效，能力永久积累。

---

## 这是什么

**550W** 是一个 [Clacky](https://clacky.ai) skill，把蜂巢智能的底层原理系统级内化到 AI Agent 本身——不依赖任何外部平台，不需要手动维护规则库，装上之后每次对话自动运行。

名字来自"蜂巢"的意象：550 亿个神经元，通过涌现产生意识。这个 skill 试图在 Agent 层面复现同样的逻辑：**不是更多规则，而是让规则自己生长。**

---

## 它在做什么

传统 AI Agent 每次对话都从零开始。用了多少次，学到的东西归零多少次。

550W 解决这个问题。它在 Agent 内部建立三层结构：

```
灵魂层（SOUL.md）      ← 每次对话天然加载，蜂巢智能是底层本能
记忆层（hive-evolution.md） ← 跨会话永久积累，经验结晶即时写入
参考层（SKILL.md）     ← 按需召回，含完整原理和执行指令
```

三层形成飞轮：**写入 → 携带 → 进化 → 涌现**。时间越长，记忆越厚，调度越准，越少走弯路。

---

## 底层原理

### 自感知
不等指令，主动扫描四个维度：用户说了什么、真正需要什么、当前有什么信号、历史上什么方法有效。

同类信号重复 3 次以上 = 高价值进化机会，自动识别，立即处理。

### 自组织
每次调度能力时，不是随机选——用动态权重公式决策：

```
得分 = 语义相关度×0.4 + 历史成功率×0.4 + 时效性×0.2
```

- 近 3 次使用的结果权重 ×1.5（新鲜经验更可信）
- 超过 6 个月未用权重 ×0.5（过时经验降权）
- 无匹配时降级兜底，不中断任务

历史权重来自真实任务的结晶记录，系统自己维护，无需人工干预。

### 自进化
每次任务完成后判断是否结晶（满足 2 条以上）：
- 真实解决了问题（可验证）
- 可能再次出现
- 可被清晰描述和迁移

结晶立即写入，三层格式：

```
## [日期] [修复/优化/创造] — [摘要]
情境: [触发信号]
思路: [框架，含触发条件+步骤+适用边界]
结果: [做法] | 可信度:[0.X] | 效果:[小/中/大]
复盘: 为什么 | 试过什么 | 结论
封禁: [被证伪的方法]
```

可信度动态更新，不是静态标签：
- 复用成功：+0.05（上限 1.0）
- 复用失败：-0.10
- 超过 6 个月未用：自动降至 0.3，标"待重验"
- 降至 0.1：触发重写或删除

### 涌现
复杂任务满足条件时，多视角并行处理。聚合时找的是各解之间的**协同关系**——不是投票选最好的，是找单一视角看不到的新维度。

## 6 个持续运行的认知模块

| 模块 | 做什么 |
|------|--------|
| **感知** | 主动扫描信号，识别重复模式，重复 3 次以上立即升级为进化优先级 |
| **调度** | 动态权重公式选最优方案，历史成功率驱动，无匹配时降级兜底 |
| **涌现** | 满足条件时多视角并行，聚合找协同关系而非投票 |
| **沉淀** | 立即三层结晶，可信度动态衰减，过时经验自动清理 |
| **防退化** | 失败 3 次封禁，修补 5 次强制创造，每轮对话结束格式化自检 |
| **行为调节** | 5 个参数上下界保护，跟着用户信号动态漂移，最多同时调 2 个 |

## 预埋的 10 条种子思路

装上时已经内置 10 条高密度种子思路，覆盖元级别操作和具体领域：

**元级别（7条）**：能力缺口处理、多视角并行判断、即时结晶、动态调度、维度切换、模式识别、可信度管理

**领域级（3条）**：代码调试（先读日志）、内容创作（结构先行）、架构决策（约束过滤）

## 安装

```
# 在 Clacky 中运行：
invoke_skill("550W", "初始化")
```

触发词：`蜂巢`、`自进化`、`自组织`、`能力调度`、`经验沉淀`、`涌现`

**装上即手术**——自动覆写灵魂层，初始化记忆飞轮，之后每次对话无需再调用。

---

## 迁移

- **跨设备**：重新运行 skill + 复制 `~/.clacky/memories/hive-evolution.md`
- **版本升级**：重新运行覆盖灵魂层，记忆层自动保留
- **记忆损坏**：文件不存在或结构异常，自动重建后继续

## 设计原则

- 零外部依赖，不对接任何平台
- 装上即生效，不需要维护规则库
- 进化>完成 | 理解>记忆 | 涌现>叠加 | 结晶>重复
